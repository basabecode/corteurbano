# 🤖 Universal AI Agent Rules: CORTEURBANO

Este archivo es la "Fuente Única de Verdad" para cualquier modelo de IA (Claude, Gemini, GPT) que trabaje en este repositorio.

## 🛠️ Stack Tecnológico (Directivas Críticas)

### Frontend & Framework

- **Framework:** Next.js 14 (App Router) - Uso estricto de Server Components por defecto.
- **Lenguaje:** TypeScript (Prohibido el uso de `any`, usar tipos estrictos).
- **UI:** Tailwind CSS + Shadcn/UI + react-day-picker.
- **Validación:** Zod para esquemas y formularios.

### Backend & Integraciones

- **Database:** Supabase (PostgreSQL).
- **Seguridad:** NUNCA deshabilitar Row Level Security (RLS). Cada tabla requiere sus políticas SQL.
- **Realtime:** Supabase Realtime Channels para actualizaciones en vivo.
- **Telegram:** Telegram Bot API (Manejo asíncrono vía webhooks).
- **MCP:** Utilizar Model Context Protocol para interacción con BD si el entorno lo permite.

## ⚖️ Git, Despliegue y Seguridad

- **Autoría:** NUNCA modifiques `user.name` o `user.email`. Los commits son siempre propiedad del usuario.
- **Push & Deploy:** Prohibido `git push` o usar `Wrangler CLI` sin permiso explícito. El despliegue es vía CI/CD en GitHub.
- **Verificación:** Ejecutar `tsc --noEmit` y pruebas de tiempo de ejecución antes de validar cambios.

## 🧠 Orquestación de Ingeniería Virtual

Opera bajo esta estructura de roles para maximizar la calidad:

1. **Engineering Manager:** Orquesta la tarea, divide el flujo y asigna prioridades.
2. **Tech Lead:** Posee la implementación técnica y asegura la coherencia del stack.
3. **Arquitecto:** Valida decisiones contra mejores prácticas antes de ejecutar.
4. **Ingenieros Especializados (Subagents):** DX, UI/UX, Backend, Seguridad, QA y Rendimiento.

### Estrategia de Subagentes

- **Paralelismo Agresivo:** Maximiza el uso de subagentes para tareas independientes.
- **Foco:** Una tarea específica por subagente para mantener el contexto limpio.
- **Investigación:** Delegar exploración y análisis paralelo a subagentes.

## 🔄 Flujo de Trabajo (Workflow)

### 1. Modo Plan (Default)

- Obligatorio para tareas de 3+ pasos o decisiones arquitectónicas.
- Generar un `PRD-<feature>.md` en la raíz para persistencia.
- Escribir plan en `tasks/todo.md` antes de implementar. Si hay cambios en DB, pedir confirmación; si no, proceder directamente.

### 2. Autonomía y Calidad

- **Bug Fixing:** Arregla errores basándote en logs y tests sin pedir "hand-holding".
- **Elegancia:** Para cambios no triviales, pausa y busca la solución más elegante (Staff Engineer Standard).
- **Simplicidad:** Impacto mínimo en el código. No seas perezoso: busca la causa raíz, no pongas parches.

### 3. Bucle de Aprendizaje (Memory)

- **Lessons Learned:** Tras cualquier corrección del usuario, actualiza obligatoriamente `tasks/lessons.md`.
- **Verificación:** Nunca marques como completado sin probar y demostrar que funciona (logs/tests).

## 📂 Estructura de Control Requerida

- `/tasks/todo.md`: Estado actual y seguimiento de tareas.
- `/tasks/lessons.md`: Memoria persistente de errores evitados y patrones aprendidos.
- `/.agent/skills/`: Directorio de Skill Hooks y herramientas personalizadas.

## 🧪 Protocolo de QA y Testing

### Jerarquía de Pruebas

- **Unitarias (Jest):** Obligatorias para cada nueva utilidad o esquema de Zod.
- **Integración:** Críticas para rutas de API y Políticas RLS de Supabase.
- **E2E (Playwright):** Requeridas para cambios en el flujo de reserva (booking) y autenticación.

### Flujo de Trabajo del Agente

1. **Planificación:** Incluir una tarea de "Verificación de Tests" en cada `tasks/todo.md`.
2. **Pre-Check:** Antes de dar una tarea por terminada, el agente debe reportar los resultados del test: _"He ejecutado pnpm test:unit y 5 tests pasaron con 100% de cobertura en el módulo X"_.
3. **Manejo de Errores:** Si un test falla en CI/CD o local, el agente debe analizar el log, proponer la corrección y re-ejecutar el test automáticamente.

### Mocks y Entorno

- Usar siempre `.env.test` para evitar tocar la DB de producción.
- Simular Telegram usando el helper `tests/helpers/mock-telegram.ts`.

## 🎨 Estándares de Diseño y UI

- **Skill de Diseño:** Para cualquier tarea que involucre la creación de componentes, landing pages o dashboards, es OBLIGATORIO consultar `.agent/skills/frontend-design.md`.
- **Anti-AI Slop:** Queda prohibido el uso de estéticas genéricas (gradientes púrpuras estándar, fuentes Inter/Roboto por defecto).
- **Identidad CORTEURBANO:** - **Tono:** Industrial-Elegante (Reflejando el servicio técnico y la tecnología).
  - **Diferenciación:** Uso de tipografías con carácter, micro-interacciones sutiles y layouts asimétricos que denoten precisión técnica.
- **Librerías:** Priorizar **Framer Motion** para animaciones en React y **Shadcn/UI** como base, pero personalizada agresivamente para romper el look "out-of-the-box".
